## <a name="alertlogic-log-manager"></a>Správce protokolu AlertLogic
Správce protokolu výstrahy logiku shromažďuje virtuálního počítače, aplikace a platformy Azure protokolů pro analýzu zabezpečení a jejich uchovávání. Shromažďuje taky protokol činnosti Azure prostřednictvím rozhraní API služby Azure monitorování. Tyto informace slouží ke zjištění malfeasance a splňovat požadavky na dodržování předpisů.

[Přejděte na dokumentaci.][alertlogic-doc]

## <a name="appdynamics"></a>AppDynamics
AppDynamics Application Performance Management (APM) umožňuje vlastníkům aplikace rychle vyřešit kritická místa výkonu a optimalizovat výkon svých aplikací běžících v prostředí Azure. Může monitorovat Azure Cloud Services (PaaS), web a pracovní role virtuálních počítačů (IaaS), vzdálené služby detekce (Microsoft Azure Service Bus), Microsoft Azure Queue, vzdálené služby Microsoft Azure (Azure Blob), Azure Queue (Microsoft Service Bus), Data Úložiště a úložiště blobů Microsoft Azure. AppDynamics APM je k dispozici v Azure Marketplace.

[Přejděte na dokumentaci.][appdynamics-doc]

## <a name="circonus"></a>Circonus
Circonus je mikroslužeb monitorování a analytické platformě vytvořené pro místní nebo SaaS nasazení. Je plně automatable rozhraní API na střed platforma větší škálovatelnost a spolehlivější než systémy sleduje. Vytvořených pro požadavky DevOps, Circonus zajišťuje výstrahy na základě percentilu, grafy, řídicí panely a intelligence strojové učení, který povolit optimalizace podnikání. Circonus monitoruje vaše cloudové prostředky Microsoft Azure a jejich využití v reálném čase. Circonus můžete použít ke shromažďování a sledovat metriky pro proměnné, které chcete k měření prostředků a aplikací. S Circonus získáte přehled systémové provozní stav, výkon aplikace a využití prostředků Azure.

[Přejděte na dokumentaci.][circonus-doc]

## <a name="cloudhealth"></a>CloudHealth

CloudHealth poskytuje viditelnosti, intuitivní optimalizace a postupy pevných zásad správného řízení pro správu cloudu. Platforma Cloudhealth umožňuje podnikům a MSPs a maximalizovat tak návratnost investic do cloudu. Jisti rozhodnutí ohledně náklady, využití, výkonu a zabezpečení.

## <a name="cloudyn"></a>Cloudyn
Cloudyn spravuje a optimalizuje více platformami, hybridního cloudu nebo nasazení pomohou podniky plně Uvědomte si, jejich potenciální cloudu. Řešení SaaS nabízí přehled o využití, výkonu a nákladů. Poskytuje přehled a řešitelné doporučení pro inteligentní řízení optimalizace a cloudu. Cloudyn umožňuje odpovědnosti za přesné vrácení peněz a správě přidělení hierarchické náklady. Cloudyn jsou integrované s Azure Monitoring k poskytnutí přehledy a řešitelné doporučení k optimalizaci nasazením Azure.

[Přejděte na dokumentaci.][cloudyn-doc]

## <a name="datadog"></a>Datadog
Datadog je světově úvodní sledování služby pro cloudové škálování aplikace. Spojuje dat ze serverů, databází, nástroje a služby k dispozici jednotný pohled na celý zásobník. Tyto možnosti jsou k dispozici na platformu analýzy dat založené na SaaS. Tato služba umožňuje vývojářů a Ops týmů a pracujte společně výpadky, vyřešte problémy s výkonem a zajistit, aby vývoj a nasazení cykly dokončen včas. Díky integraci Datadog a Azure, můžete shromažďovat a zobrazovat metriky z vaší infrastruktury. Korelovat metriky virtuálního počítače s metriky na úrovni aplikace. Vyfiltrování a rozčlenění vaše metriky pomocí libovolné kombinace vlastností a vlastní značky.

[Přejděte na dokumentaci.][datadog-doc]

## <a name="dynatrace"></a>Dynatrace
Dynatrace OneAgent se integruje s virtuálními počítači Azure a aplikační služby prostřednictvím mechanismu rozšíření Azure. Tímto způsobem Dynatrace OneAgent může shromažďovat metriky výkonu o hostitele, sítě a služby. Kromě zobrazení právě metriky, vizualizuje Dynatrace prostředí začátku do konce. Zobrazuje transakcí ze strany klienta v databázové vrstvě. Dynatrace poskytuje korelace na základě AI problémů a nabízí plnou integraci kořenové analýzu příčina umožnit metoda úrovně přehledy kód a databáze. Tento přehled jednodušší optimalizace řešení potíží a výkonu.

[Přejděte na dokumentaci.][dynatrace-doc]

## <a name="logic-monitor"></a>Logiku monitorování
LogicMonitor® je přední založené na SaaS, sledování výkonu platforma pro komplexní infrastruktury IT. S pokrytí pro tisíce technologie LogicMonitor poskytuje podrobné přehled o výkonu infrastruktury a aplikace. LM cloudu komplexní Azure monitorování umožňuje uživatelům korelovat výkon cloudu Azure, místní a hybridní cloudové prostředky, všechny z jedné platformy. Automatizované prostředek zjišťování, součástí monitorování šablony, předkonfigurované prahových hodnot výstrah a přizpůsobit řídicí panely kombinace umožnit IT rychlosti, flexibilitu a viditelnost potřebné k úspěšné.

[Přejděte na dokumentaci.][logicmonitor-doc]

## <a name="moogsoft"></a>Moogsoft
Moogsoft AIOps zrychluje agilní obchodní transformace.

Nástroje Microsoft Azure automatizace a řízení zadejte okno v reálném čase do stavu aplikace a mikroslužeb nasazené v Azure a pomoci orchestraci diagnostiky a sady runbook pro rychlejší nápravu.
Další nástroje třetích stran zadejte okno na místní aplikace a stav infrastruktury.

Moogsoft AIOps automatizuje událost postup nápravy beze změny existujících procesů a organizační struktury. 

Spuštění ve vašem Azure-nemovitosti, s integrací nástroje monitorování a automatizace napříč hybridní topologie fabric, Moogsoft AIOps aktivně zjistí aplikace, které mají vliv incidenty dříve, dynamicky orchestruje s příslušnými prostředky tak, aby byly situaci clustery a snižuje střední čas k nápravě a negativní dopad na zkušeností zákazníků. 

[Přejděte na dokumentaci.][moogsoft-doc]

## <a name="opsgenie"></a>OpsGenie
OpsGenie funguje jako dispečera pro výstrahy generované Azure. OpsGenie Určuje správné lidí, kteří budou na základě plány na volání a eskalací. Můžete upozorněním pomocí e-mailem, textové zprávy (SMS), telefonního hovoru nebo nabízená oznámení. Azure generuje výstrahy pro zjištěných problémů. OpsGenie zajistí, že příslušní lidé pracují na problém.

[Přejděte na dokumentaci.][opsgenie-doc]

## <a name="pagerduty"></a>PagerDuty
PagerDuty, špičkové řešení správy incidentů poskytl prvotřídní podporu pro Azure výstrahy o metrikách. PagerDuty podporuje oznámení na Azure monitorování výstrahy, oznámení o automatickém škálování, protokolu událostí auditu a metriky úrovní platformu pro služby Azure. Tato vylepšení udělení vyšší přehled základní platformu Azure. Mohou využít výhod funkcí správy incidentů na PagerDuty pro odpověď v reálném čase. Rozšířená integrace Azure je k dispozici prostřednictvím webhooky. Webhooky umožňují instalaci a nastavení řešení snadno a rychle.

[Přejděte na dokumentaci.][pagerduty-doc]

## <a name="sciencelogic"></a>ScienceLogic
ScienceLogic přináší nové generace IT služby záruku platforma pro správu jakékoli technologie kdekoli. ScienceLogic poskytuje škálování, zabezpečení, automatizaci a odolnosti, které jsou nezbytné ke zjednodušení úloh spojených se správou IT prostředky, služeb a aplikací. Platforma ScienceLogic používá rozhraní API služby Azure na rozhraní s Microsoft Azure. ScienceLogic vám v reálném čase přehled služby Azure a prostředky. Abyste věděli, když něco nefunguje a můžete je vyřešit je rychlejší. Můžete také spravovat Azure spolu s vašimi cloudů a datových center systémy a službami.

## <a name="signalfx"></a>SignalFx
SignalFx je přední v reálném čase provozní intelligence pro datové DevOps. Služba zjistí a shromažďuje metriky mezi všechny komponenty v cloudu, nahraďte tradiční bod nástrojů a poskytování v reálném čase přehled dnešních dynamických prostředích. Využití nesmírně škálovatelná SignalFx platformy, platformy SaaS je optimalizovaná pro kontejner a mikroslužeb na základě architektury a nabízí výkonné vizualizace, proaktivní upozorňování a možnosti spolupráce třídění napříč organizací všech velikostí. SignalFx integruje přímo s Azure monitorování, a také prostřednictvím konektory s otevřeným zdrojem, jako je například Telegraf, statsD a collectd zajistit nejvhodnější třída řídicí panely, analýzy a výstrahy pro Azure.

[Přejděte na dokumentaci.][signalfx-doc]

## <a name="signl4"></a>SIGNL4
SIGNL4 - výstrahy mobilní aplikace pro operace týmy - je nejrychlejší způsob, jak směrování kritické výstrahy z Azure monitorování odpovídajícím lidem ve správný čas – kdekoli voláním push, textu a hlasu. SIGNL4 spravuje povinností na volání a posuny týmu, sleduje doručení a vlastnictví výstrah a eskaluje v případě potřeby. Je k dispozici plnou průhlednost ve vašem týmu. Pomocí velice snadné REST web hák z SIGNL4 můžete libovolnou službu Azure připojen k žádné úsilí. S SIGNL4 uvidíte až 10 x rychlejší reakci prostřednictvím e-mailová oznámení a ruční výstrahy.

## <a name="sumo-logic"></a>Sumo logiky
Sumo logiku je služba analýzy dat zabezpečené, cloudu nativní, počítače, přenosu v reálném čase, průběžné intelligence z strukturovaná, částečně strukturovaná i nestrukturovaná data prostřednictvím zásobníku a životního cyklu celou aplikaci. Více než 1 000 zákazníky po celém světě spoléhají na Sumo logiku pro analýzy a statistiky k vytvoření, spuštění a zabezpečení svých aplikací a cloudových infrastruktur. Pomocí Sumo logiku zákazníkům získat více klientů, model služby výhod a pomáhá tak zvýšit růst, konkurencí a obchodní hodnotu.

[Víc se uč.][sumologic-doc]

## <a name="turbonomic"></a>Turbonomic
Turbonomic přináší automatizace pracovního vytížení pro cloudy, hybridní pomocí současně optimalizace výkonu, náklady a dodržování předpisů v reálném čase. Turbonomic pomáhá organizacím být elastické v jejich Azure majetku podle nepřetržitě optimalizace majetku zajistit, že aplikace neustále získají prostředky, které potřebují k poskytování jejich SLA a nic další napříč výpočty, úložiště a sítě pro IaaS a PaaS vrstva. Organizace můžete simulovat migrace, správně škálování zatížení a vyřazení prostředky datacentra jeho zodpovědné migrace na Azure v době, v rámci rozpočtu, přičemž musí být zajištěno výkon a dodržování předpisů. Turbonomic je založený na rozhraní API a běží jako virtuální počítač bez agentů v Azure i v místním prostředí.

[Víc se uč.][turbonomic-doc]
