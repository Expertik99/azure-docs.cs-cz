# <a name="azure-functions-and-serverless-platform-security"></a>Azure Functions a zabezpečení bez serveru platformy
## <a name="abstract"></a>abstraktní
Většina firem potřebovat významné množství prostředků a času ke správě serverů, který přidá náklady. Pokud podniky může používat méně prostředků ke správě serverů, budou se zaměřit na vytváření kvalitních aplikací.  

Bez serveru computing pomáhá přesně k tomu, protože je spravovaná infrastrukturu, která budete muset spustit a škálování aplikací. Architektura bez serveru představuje abstrakci serverů, infrastruktury a operačních systémů. Bez serveru výpočty vycházejí z reakce na události a aktivačních událostí, které jsou všechny probíhající v near v reálném čase – v cloudu. 

Jako plně spravovaná služba Správa serveru a plánování kapacity jsou neviditelná pro vývojáře. Bez serveru framework usnadňuje vývoj a nasazení aplikace bez serveru pomocí Azure Functions. Je rozhraní příkazového řádku (CLI), která nabízí strukturu a automatizace, které vám umožní vytvořit sofistikované, založeného na událostech, bez serveru architektury skládá z funkce a události. Azure funkce je samostatná jednotka nasazení, jako je mikroslužby. Je to jenom kód nasazen v cloudu, nejčastěji zapsaná provést jednu úlohu.

Bez ohledu výhody bez serveru zabezpečení má svou vlastní rizikové faktory, jak nakládat s. Bez serveru přístup nemá zavést nové otázky zabezpečení, ale vyžaduje to má přístup k existující z důvodů zabezpečení. Tento dokument se zaměřuje na tyto bezpečnostní otázky: 
* Výhody bez serveru platformy
* Problémy se zabezpečením počítačů bez serveru
* Problémy zabezpečení a jejich zmírnění v kontextu systému Azure
* Zabezpečení bez serveru platformy Microsoft
